AIRS 로직 Flow와 UI 연결

AIRS 앱의 실제 사용 흐름을 화면 단위로 해석한 내용입니다. 팀원 리뷰 시 “예쁜 화면”보다 각 화면이 어떤 상태와 다음 행동을 담당하는지 함께 봐주세요.

1

사용자 진입과 인증

사용자는 로그인 또는 회원가입 화면에서 계정 상태를 확정합니다. 인증이 끝나면 앱은 사용자의 등록 노드 목록을 불러오는 흐름으로 넘어갑니다.

관련 UI: 01 로그인, 02 회원가입
2

노드 목록 분기

등록된 노드가 있으면 온도, 습도, 연결 상태, 신호 강도, 최근 업데이트를 카드로 보여줍니다. 없으면 새 디바이스 추가를 중심으로 빈 상태를 안내합니다.

관련 UI: 03 내 노드 있음, 04 내 노드 없음
3

Bluetooth 검색과 페어링

새 디바이스 추가를 누르면 주변 AIRS 기기를 검색합니다. 사용자가 기기를 선택하면 연결 중 모달을 보여주고, 결과에 따라 성공 또는 실패 상태로 분기합니다.

관련 UI: 05 Bluetooth 검색, 06 연결 중, 07 실패, 08 성공
4

Wi-Fi 정보 전달

Bluetooth 페어링이 성공하면 선택된 AIRS 노드에 Wi-Fi 이름과 비밀번호를 전달합니다. 이 화면은 입력 실수와 네트워크 조건 안내가 특히 중요합니다.

관련 UI: 09 Wi-Fi 설정
5

설정 완료 또는 복구

Wi-Fi 연결이 성공하면 내 노드 또는 노드 상세로 이동합니다. 실패하면 비밀번호, 공유기 거리, 재시도 흐름을 안내해 사용자가 회복할 수 있게 해야 합니다.

관련 UI: 10 설정 성공, 11 설정 실패
6

운영 중 모니터링

등록 이후에는 노드 상세에서 센서값과 기기 정보를 확인합니다. 온도 조절 대안들은 이 상세 화면을 더 제어 중심으로 확장할 때 참고할 수 있습니다.

신규 기능 회의 정리

2026-05-24 팀원 회의에서 정리한 신규 기능 아이디어입니다. 기존 온습도 표시와 예측 기능을 유지하면서, 적외선 제어, 외부 온도 비교, 부재중 온도 보호, GPS 기반 자동 활성화로 앱 사용성을 확장하는 방향입니다.

예측 대시보드 현재 온습도와 20분 후 예측 온도를 함께 보여주고, LCD 표시 정보를 앱에서도 확인합니다.
외부 온도 비교 기상청 API 기반 외부 온도와 실내 온도를 비교해 냉방 판단을 돕습니다.
부재중 온도 보호 mmWave 인체 감지를 활용해 사람이 없을 때 방 온도 보호 규칙을 설정합니다.
GPS 자동 활성화 집 근처 진입 또는 외출 상태를 감지해 에어컨 켜기/끄기 자동화를 제안합니다.

디자인 옵션 장단점

아래 평가는 현재 이미지 기준의 빠른 판단입니다. 최종 선택 전에는 Android 구현 난이도, 팀 디자인 취향, 실제 기능 범위와 함께 다시 확인하면 좋습니다.

현재안: Friendly Mascot

밝은 스마트홈 분위기와 펭귄 브랜드 요소가 가장 직접적으로 보이는 기준안입니다.

장점첫인상이 친근하고, 노드 목록/상태/추가 버튼이 직관적입니다.
단점상단 이미지와 큰 CTA가 반복되면 실제 관리 앱보다는 홍보 이미지처럼 보일 수 있습니다.

로그인 대안: Night Premium

어두운 배경, 빛나는 공기 흐름, 글래스 카드로 프리미엄 이미지를 강조합니다.

장점브랜드 인상이 강하고, 팀원에게 “기존안과 다른 방향”을 보여주기 좋습니다.
단점전체 앱을 다크 톤으로 밀지 않으면 다른 화면과 일관성 조정이 필요합니다.

로그인 대안: Fresh Card

밝은 생활 이미지와 명확한 로그인 카드로 가장 안전한 온보딩 구성을 보여줍니다.

장점가독성과 사용성이 좋고, 현재안보다 정보 구조가 차분합니다.
단점현재 밝은 방향과 크게 멀지는 않아 강한 대안성은 약할 수 있습니다.

내 노드 대안: Summary Grid

상단 요약 지표와 2열 카드 배치를 사용해 관리 화면에 가까운 구조를 만듭니다.

장점총 노드, 연결됨, 불안정, 평균 신호가 한눈에 보여 운영 화면으로 설득력이 있습니다.
단점이미지와 요약, 카드가 함께 있어 화면 밀도가 높아질 수 있습니다.

내 노드 대안: Control Dashboard

대표 노드를 크게 보여주고, 나머지 노드를 칩/카드로 전환하는 제어형 패턴입니다.

장점온도/습도/모드 같은 핵심 정보가 강하게 보이고, 앱의 차별성이 큽니다.
단점현재 요구가 “노드 목록 관리”라면 특정 노드 제어 화면처럼 오해될 수 있습니다.

온도 조절 대안: Gauge Control

큰 원형 온도 게이지와 상태 패널을 중심으로 한 제어 중심 화면입니다.

장점온도 조절 기능을 추가할 경우 가장 강한 시각적 기준점이 될 수 있습니다.
단점현재 백엔드/앱 범위가 모니터링 중심이면 기능이 과해 보일 수 있습니다.

온도 조절 대안: Trend Detail

온도 추이 그래프와 평균/최저/최고 값을 보여주는 분석형 상세 화면입니다.

장점센서 데이터 앱이라는 정체성이 분명하고, 노드 상세 화면과 잘 맞습니다.
단점첫 화면으로 쓰기에는 시각적 재미가 적고, 데이터가 적으면 빈약해 보일 수 있습니다.

온도 조절 대안: Cozy Wellness

따뜻한 색감과 카드형 제어를 조합해 생활감 있는 스마트홈 화면을 만듭니다.

장점딱딱한 IoT 앱보다 사용자 친화적이고, 공기/쾌적함 이미지가 강합니다.
단점AIRS의 블루-그린 정체성과 맞추려면 색상 조정이 필요합니다.

온도 조절 대안: Hybrid Panel

상단 다크 제어 영역과 하단 밝은 패널을 나눠 기능을 많이 담는 방향입니다.

장점게이지, 상태, Wi-Fi 설정, 필터 정보까지 확장성이 좋습니다.
단점한 화면에 기능이 많아 MVP 단계에서는 복잡하게 느껴질 수 있습니다.

팀원 리뷰 포인트

  • 흐름이 자연스러운가? 로그인부터 노드 등록, Bluetooth, Wi-Fi, 성공/실패 상태까지 이어지는지 확인합니다.
  • 정보 우선순위가 맞는가? 온도, 습도, 연결 상태, 신호 강도, 업데이트 시간이 한눈에 들어오는지 봅니다.
