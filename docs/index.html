AIRS 로직 Flow와 UI 연결

앱의 실제 상태 전환을 화면 단위로 연결했습니다. 신규 기능은 기존 노드 상세 이후의 확장 흐름으로 보되, 예측과 자동화가 메인 화면에 어느 정도 노출되어야 하는지도 함께 검토합니다.

1

사용자 진입과 인증

사용자는 로그인 또는 회원가입 화면에서 계정 상태를 확정합니다. 인증 완료 후 등록 노드 목록을 불러옵니다.

관련 UI: 01 로그인, 02 회원가입
2

노드 목록 분기

등록된 노드가 있으면 온습도, 연결 상태, 신호 강도, 최근 업데이트를 보여주고, 없으면 새 디바이스 추가를 안내합니다.

관련 UI: 03 내 노드 있음, 04 내 노드 없음
3

Bluetooth 검색과 페어링

새 디바이스 추가 후 주변 AIRS 기기를 검색합니다. 선택한 기기의 연결 중 상태와 성공/실패 결과를 분명하게 보여줍니다.

관련 UI: 05 Bluetooth 검색, 06 연결 중, 07 실패, 08 성공
4

Wi-Fi 전달과 복구

Bluetooth 연결 성공 후 노드에 Wi-Fi 정보를 전달합니다. 성공 시 목록/상세로 이동하고, 실패 시 재시도 가능한 원인을 안내합니다.

관련 UI: 09 Wi-Fi 설정, 10 설정 성공, 11 설정 실패
5

운영 중 모니터링

등록 이후 노드 상세에서 현재 온습도, 연결 상태, 기기 정보를 확인합니다. 이 화면이 신규 기능의 진입점이 됩니다.

관련 UI: 12 노드 상세
6

예측과 외부 비교

현재 센서값과 기상 API를 함께 사용해 20분 후 온습도와 외부 대비 온도 차이를 표시합니다.

관련 UI: 13 예측 대시보드, 14 외부 온도 비교
7

부재중 보호 자동화

사람 감지 센서와 보호 온도 기준을 조합해 방이 너무 더워질 때 자동으로 에어컨 동작 여부를 제안하거나 실행합니다.

관련 UI: 15 부재중 온도 보호
8

위치 기반 자동 활성화와 제어

사용자가 집 근처에 들어오면 에어컨을 미리 켜고, 외출 또는 부재가 감지되면 끄기 알림을 제공합니다. 온도 조절 화면은 이 제어 흐름의 후보입니다.

관련 UI: 16 GPS 자동 활성화, 17 온도 조절 페이지 대안

신규 기능 회의 정리

2026-05-24 팀원 회의에서 정리한 아이디어입니다. 기존 LCD 온습도 표시와 예측 기능을 유지하면서, 외부 센서·기상 API·사람 감지·GPS 자동화를 AIRS의 차별점으로 확장하는 방향입니다.

UI 번호 추가 UI 기능 목적
13 예측 대시보드 향후 20분 후 온습도 예측값을 보여주고, 현재 온도와 예측 온도를 비교해 에어컨을 미리 켤지 판단합니다.
14 외부 온도 비교 기상청 API 또는 위치 기반 외부 온도를 가져와 실내 온도와 비교하고, 바깥 날씨 대비 방이 얼마나 더운지 보여줍니다.
15 부재중 온도 보호 사람이 없을 때도 방 온도가 너무 높아지면 식품이나 민감한 물품 보호를 위해 에어컨 동작 여부를 제안하거나 자동 제어합니다.
16 GPS 자동 활성화 사용자가 집 근처 일정 거리 안에 들어오면 에어컨을 자동으로 켜고, 집 밖으로 나가면 끄기 알림을 제공합니다.
17 온도 조절 페이지 대안 노드 상세 이후 확장될 수 있는 직접 제어 화면 후보로, 온도 게이지와 모드·풍량·냉방 조작 패턴을 비교합니다.
13

20분 후 온습도 예측

현재 센서값과 외부 온도를 함께 보고, 금방 더워질지 미리 판단해 에어컨 실행 여부를 결정합니다.

14

외부 온도 대비 온도 차이

기상청 API와 위치 설정을 활용해 실내외 온도 차이를 보여주고, 사용자가 더위를 체감하기 전에 판단을 돕습니다.

15

부재중 온도 보호

사람이 없을 때도 방 온도가 너무 올라가면 식품이나 민감한 물품 보호를 위해 에어컨 동작을 제안합니다.

16

GPS 자동 활성화

사용자가 집 근처에 들어오면 자동으로 에어컨을 켜고, 외출 상태가 길어지면 끄기 알림을 제공합니다.
